# Introduction
Data Mesh is a decentralized data architecture where data is owned and managed by domain teams, and exposed as “data products”, rather than managed by a single team.
# The 4 key principles
## 1. Domain-oriented ownership
- Sales owns sales data
- Finance owns finance data
- Marketing owns marketing data

No central bottleneck team.
## 2. Data as a product
Each dataset must be:
- well-documented
- reliable
- discoverable
- versioned
## 3. Self-serve data platform
A central platform team provides:
- infrastructure (storage, compute)
- pipelines tooling
- governance frameworks

But not the data itself.
## 4. Federated computational governance
- Global rules (security, privacy, standards)
- Locally implemented by domain teams

 Combines autonomy + consistency.
