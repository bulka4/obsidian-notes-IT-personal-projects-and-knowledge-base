# Introduction
Schema validation is a practice of checking a correctness of new data. In contrast to data quality tests ([[dbt benefits - Data quality tests (unit tests)|link]]), it focuses on data structure rather than whether values are correct.

For example, schema validation checks if there are:
- Missing columns
- Wrong data types
- Correct formats (e.g., date format)
