# Introduction new
In this project, we:
- Deploy a MLflow Tracking server on Kubernetes (AKS)
- Run a MLflow project which:
	- Prepares a training dataset
	- Trains and evaluates ML model
	- Saves the workflow’s metadata and artifacts: created model, hyperparameters used and evaluation metrics

Infrastructure is prepared using Terraform and Helm:
- Terraform creates AKS and all the other cloud resources (in Azure) needed
- Using Helm we deploy a MLflow Tracking server and run a MLflow project

**Outcome:**
After deploying MLflow Tracking Server, any MLflow project files can be uploaded to Azure File share and run on Kubernetes using a Helm chart.

After running a project, saved metrics and artifacts can be reviewed on the MLflow Tracking Server website.
